# Python_GUI_Hospital_Appointment_APP



# Bu projenin temel amacı python ile bir arayüz oluşturmaktır. Bu arayüzün temel amacı hastane randevu sistemini simule etmektir. Bu sistem temelde kullanıcıya belirli seçenekler sunarak belirli tarih ve saatte randevu oluşturabilmesini sağlar. Daha detaylı bakacak olursak sisteme tanımlı 3 farklı departman bulunmaktadır. Bunlar kardiyoloji, nöroloji ve göz hastalıklarıdır. Her departman için toplamda 3 adet hastane bulunmaktadır. Ve her hastane ve her departman için 2 farklı doktor sisteme tanımlanmıştır. Yani toplamda 12 farklı doktor bulunmaktadır. Sistem çalıştırıldığında tüm seçenekler boş olarak gelmektedir.  Kullanıcı ilk olarak randevu almak istediği departmanı seçer. Daha sonra kullanıcı hangi hastaneden randevu almak istediğini seçer. Seçilen departman ve hastaneye göre kullanıcıya doktor bölümünde özel 2 doktor gösterilir. Kullanıcı bunu da seçtikten sonra isim ve soy isim girmesi gerekmektedir. Kullanıcı bu işlemi de gerçekleştirdiğinde bir tarih seçmesi gerekir. Tarih seçmek için bir takvim açılır ve kullanıcı bu ekrandan kolaylıkla tarihi seçebilir. Daha sonra ise yine sisteme tanımlanan saat 8.00 dan saat 16.00 a kadar 30 ar dakika aralıklarla saatler ekrana gösterilir ve kullanıcının seçimine sunulur. Kullanıcı tüm bilgileri girdiğinde randevu oluştur butonuna asar. Ekranda randevunun başarıyla oluşturulduğuna dair bir bilgi mesajı görülecektir. Bu bilgi mesajından sonra yine sistem üzerine entegre edilen listbox üzerinde oluşturulan randevu görüntülenebilmektedir. Bu görüntüleme ekranında isim-soy isim-bölüm-hastane-doktor-tarih-saat şeklinde bilgiler gösterilir. Eğer ki kullanıcı herhangi bir alanı doldurmadan randevu oluşturmak ister ise sistem uyarı verecek ve randevuyu onaylamayacaktır. Kullanıcı randevuyu silmek istediğinde listbox bölümünden randevuyu seçmeli de delete butonuna basmalıdır. Bu işlem de gerçekleştiğinde yine kullanıcıya bir bilgi mesajı gösterilmektedir. Eğer ki kullanıcı herhangi bir randevuyu seçmeden bu butona basarsa yine kullanıcıya hata mesajı gösterilmektedir. Tüm bu işlemler arka planda bir txt dosyasına kaydedilip dosya üzerinden gerçekleştirilmektedir. Dolayısıyla sistem kapatıldığında randevular kaybolmamaktadır. 



# The main purpose of this project is to create an interface with python. The main purpose of this interface is to simulate the hospital appointment system. This system basically provides the user with certain options to create an appointment at a certain date and time. If we look in more detail, there are 3 different departments defined in the system. These are cardiology, neurology and eye diseases. There are 3 hospitals in total for each department. And 2 different doctors are defined in the system for each hospital and each department. So there are 12 different doctors in total. All options are blank when the system is started. The user first selects the department he wants to make an appointment with. Then the user chooses which hospital he wants to make an appointment with. According to the selected department and hospital, 2 special doctors are shown to the user in the doctor section. After selecting this, the user must enter a first and last name. When the user also performs this operation, he must select a date. A calendar opens to select the date and the user can easily select the date from this screen. Afterwards, the hours are displayed on the screen at 30-minute intervals from 8.00 to 16.00, which are also defined in the system, and presented to the user's choice. When the user enters all the information, he hangs on the create appointment button. An information message will appear on the screen that the appointment has been successfully created. After this information message, the appointment created on the listbox integrated into the system can be displayed. Information in the form of name-surname-department-hospital-doctor-date-time is displayed on this display screen. If the user wants to create an appointment without filling in any fields, the system will give a warning and will not confirm the appointment. When the user wants to delete the appointment, he must select the appointment from the listbox and press the delete button. When this process is performed, an information message is also displayed to the user. If the user presses this button without selecting any appointment, an error message is displayed to the user. All these processes are recorded in a txt file in the background and performed over the file. Therefore, appointments are not lost when the system is closed.



# Der Hauptzweck dieses Projekts ist es, eine Schnittstelle mit Python zu erstellen. Der Hauptzweck dieser Schnittstelle besteht darin, das Terminvergabesystem des Krankenhauses zu simulieren. Dieses System bietet dem Benutzer grundsätzlich bestimmte Optionen, um einen Termin zu einem bestimmten Datum und einer bestimmten Uhrzeit zu erstellen. Wenn wir genauer hinschauen, sind im System 3 verschiedene Abteilungen definiert. Dies sind Kardiologie, Neurologie und Augenkrankheiten. Es gibt insgesamt 3 Krankenhäuser für jede Abteilung. Und für jedes Krankenhaus und jede Abteilung sind im System 2 verschiedene Ärzte definiert. Insgesamt gibt es also 12 verschiedene Ärzte. Alle Optionen sind leer, wenn das System gestartet wird. Der Benutzer wählt zunächst die Abteilung aus, mit der er einen Termin vereinbaren möchte. Anschließend wählt der Nutzer aus, bei welchem Krankenhaus er einen Termin vereinbaren möchte. Je nach ausgewählter Abteilung und Krankenhaus werden dem Benutzer im Ärztebereich 2 Spezialärzte angezeigt. Nach Auswahl muss der Benutzer einen Vor- und Nachnamen eingeben. Wenn der Benutzer auch diese Operation durchführt, muss er ein Datum auswählen. Ein Kalender wird geöffnet, um das Datum auszuwählen, und der Benutzer kann das Datum einfach von diesem Bildschirm aus auswählen. Anschließend werden die Stunden im 30-Minuten-Takt von 8.00 bis 16.00 Uhr, die ebenfalls im System definiert sind, auf dem Bildschirm angezeigt und nach Wahl des Benutzers präsentiert. Wenn der Benutzer alle Informationen eingibt, hängt er an der Schaltfläche „Termin erstellen“. Auf dem Bildschirm erscheint eine Informationsmeldung, dass der Termin erfolgreich erstellt wurde. Nach dieser Informationsmeldung kann der erstellte Termin auf der im System integrierten Listbox angezeigt werden. Auf diesem Anzeigebildschirm werden Informationen in Form von Name-Nachname-Abteilung-Krankenhaus-Arzt-Datum-Uhrzeit angezeigt. Wenn der Benutzer einen Termin erstellen möchte, ohne Felder auszufüllen, gibt das System eine Warnung aus und bestätigt den Termin nicht. Wenn der Benutzer den Termin löschen möchte, muss er den Termin aus dem Listenfeld auswählen und die Schaltfläche „Löschen“ drücken. Wenn dieser Prozess durchgeführt wird, wird dem Benutzer auch eine Informationsnachricht angezeigt. Wenn der Benutzer diese Schaltfläche drückt, ohne einen Termin auszuwählen, wird dem Benutzer eine Fehlermeldung angezeigt. All diese Vorgänge werden im Hintergrund in einer txt-Datei aufgezeichnet und über die Datei ausgeführt. Daher gehen Termine nicht verloren, wenn das System geschlossen wird.
